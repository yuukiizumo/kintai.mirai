# 勤怠管理アプリ

福祉事業所での利用を想定して制作した、Laravel × React の勤怠管理アプリです。

日々の出退勤打刻だけでなく、業務報告、届出、有給管理、カレンダー管理、帳票PDF出力、自己管理レポートまでを一つの画面で扱えるように設計しました。

管理者と一般ユーザーで見える情報・操作できる範囲を分け、実際の事業所運用で起こりやすい「打刻漏れ」「届出との不一致」「有給残数の管理」「利用者ごとの勤務条件の違い」に対応できることを意識しています。

## 制作背景

勤怠管理は単に出勤・退勤時刻を記録するだけではなく、欠勤や遅刻の届出、業務報告、休日カレンダー、有給残日数、管理者確認など、複数の情報が絡み合います。

特に福祉事業所では、利用者ごとに勤務日数・勤務時間・通所形態・部署・業務区分が異なるため、一般的な勤怠システムでは運用に合わせづらい場面があります。

そこで、現場で必要になりそうな細かな運用をできるだけ画面上で完結できるようにし、管理者が毎日確認しやすい勤怠管理アプリとして制作しました。

## アプリの特徴

### 1. 管理者と一般ユーザーを分けた権限設計

一般ユーザーは自分の勤怠・届出・業務報告のみを確認できます。

管理者は全員分の勤怠を確認でき、さらに強管理者のみユーザー情報へアクセスできるようにしています。利用者情報を扱う画面では、権限によって操作範囲を分ける設計にしました。

### 2. 打刻と業務報告の入力漏れを防止

一般ユーザーは出勤・退勤の打刻ができます。

退勤時には、申告出勤時刻、申告退勤時刻、申告休憩時間、勤務場所、食事割合、欠食の有無を記録できます。

また、当日の業務報告が未入力の場合は退勤できないようにし、日報の記入漏れを防ぐ仕組みにしています。

### 3. 届出と勤怠状態を連動

欠勤、遅刻、早退、有給、半有給、業務対応届、変更届、介護サービス利用、勤務時間外通院など、複数種類の届出に対応しています。

届出がある場合は勤怠一覧の状態へ反映し、届出と勤怠状態が一致していない場合は背景色で分かるようにしました。

管理者チェック、サビ管チェックも用意し、提出後の確認フローまで画面上で完結できます。

### 4. 利用者ごとの細かな勤務条件に対応

ユーザーごとに、曜日別の出勤時刻・退勤時刻・休憩時間・勤務しない曜日を設定できます。

部署、業務区分、業務形態、通所上限日数、入社日、有給残日数、身長、体重、性別なども管理できるようにしました。

部署ごとの表示順やユーザーの並び順もドラッグアンドドロップで変更でき、一覧画面を現場の見やすい順番に調整できます。

### 5. カレンダー管理で休日や計画有給を制御

休日、土曜出勤、計画有給、自己管理レポート提出日、自由出勤日などをカレンダーとして管理できます。

土曜日は原則休日、土曜出勤が設定されている場合のみ出勤日として扱うなど、カレンダー設定をもとに勤怠判定が変わるようにしています。

### 6. PDF出力に対応

個別勤怠一覧から、用途別のPDFを出力できます。

- 出勤簿PDF
- 会社保管用PDF
- 業務報告PDF

申告時間を基準に時間内訳を計算し、提出・保管に使える帳票として出力できるようにしました。

## 工夫した点

### 現場の確認作業を減らすUI

勤怠一覧では、状態や背景色によって「届出がある」「届出がないのに状態が変わっている」「届出だけ残っている」といった状況が分かるようにしました。

管理者が一つひとつ詳細を開かなくても、一覧画面で異常に気づけることを意識しています。

### 入力ミスを減らすフォーム設計

部署に応じて業務区分の選択肢が変わるようにし、不要な入力を避けられるようにしました。

欠勤や有給の届出では不要な時間入力欄を非表示にするなど、届出の種類に応じて画面を切り替えています。

### 実運用を想定した例外処理

退勤ボタンの押し忘れ、休日の打刻、有給日の打刻、出勤していない状態での退勤など、実際に起こりやすい操作ミスを制御しています。

打刻取消は5分以内に制限し、誤操作には対応しつつ、後から自由に改ざんできないようにしました。

### 管理画面の情報量に対応

管理画面は機能が多くなりやすいため、タブ切り替えや遅延読み込みを取り入れ、必要な情報へ移動しやすい構成にしました。

部署別勤怠一覧では、横軸を日付、縦軸をユーザーにすることで、月単位の勤怠をまとめて確認できるようにしています。

### AIコメント生成

業務報告や自己管理レポートに対して、OpenRouter APIを使った管理者コメント生成機能を実装しました。

一度生成したコメントは固定し、画面を開くたびに内容が変わらないようにしています。

## まとめ

このアプリでは、単純なCRUDだけでなく、現場運用を想定した状態管理、権限制御、帳票出力、入力制御、API連携を実装しました。

勤怠データ、届出、有給、業務報告、カレンダー設定が相互に影響するため、ユーザーが迷わず操作できる画面設計と、管理者が確認しやすい情報整理を重視しています。
